# نظام المحطة المالي المشترك (دفاتر المحطة)

نظام ويب مالي تعاوني متعدّد المستخدمين: دخول بكلمة سرّ ثم اختيار الهوية، قاعدة بيانات
مركزية مشتركة، مزامنة لحظية، جرس تنبيهات لتغييرات الآخرين، ولوحة أدمن سرّية فيها السجل
الكامل وسلة المحذوفات والنسخ الاحتياطية.

## المتطلبات
- Node.js 22 أو أحدث (طُوّر على Node 24). يستخدم SQLite المدمج (`node:sqlite`) فلا حاجة لقاعدة خارجية.

## التشغيل محليًا
```bash
npm install
# عدّل كلمات السر في ملف .env (مهم قبل الاستخدام الفعلي)
npm start
```
ثم افتح: `http://localhost:8765`

- **كلمة سر المستخدمين** ثم تظهر قائمة الأسماء (يوسف عقة / إبراهيم عامر / هادي عامر).
- **كلمة سر الأدمن** تدخل مباشرة إلى لوحة الأدمن.
- القيم الافتراضية في `.env` (غيّرها): `USER_PASSWORD=station2026` ، `ADMIN_PASSWORD=admin-2026-secret`.

### الوصول من أجهزة أخرى على نفس الشبكة
شغّل الخادم على جهاز المحطة، واعرف عنوانه المحلي (مثل `192.168.1.20`)، ثم افتح من الأجهزة الأخرى:
`http://192.168.1.20:8765` (اسمح للمنفذ 8765 في جدار حماية ويندوز عند الطلب).

## أين أغيّر الإعدادات؟
- **كلمات السر ومنفذ التشغيل ونافذة الاستقرار**: ملف `.env`.
- **أسماء المستخدمين**: `server/config.js` ← مصفوفة `USERS` (تُزرع في القاعدة عند الإقلاع).
- **نافذة الاستقرار** `SETTLE_MS` (افتراضي 2500ms): المدة بعد توقّف الكتابة قبل تسجيل التغيير
  وبثّه للآخرين — تكبيرها يقلّل ضجيج التنبيهات، تصغيرها يجعل المزامنة أسرع.

## البيانات والنسخ الاحتياطية
- كل البيانات في `data/bills.db` (ملف SQLite واحد). انسخه للاحتفاظ بنسخة.
- لوحة الأدمن تُنشئ لقطات (snapshots) في `data/backups/` (يدوية + لقطة يومية تلقائية)، ويمكن
  استرجاع أي لقطة بالكامل (تُؤخذ نسخة أمان من الحالة الراهنة قبل الاسترجاع).
- لا حذف نهائي: الحذف ناعم → سلة المحذوفات في لوحة الأدمن → استرجاع فردي.

## النشر على الإنترنت
هذا **تطبيق خادم Node** (وليس موقعًا ثابتًا) — يحتاج استضافة تشغّل Node. أفضل طريقة عبر Git أو Docker:

**الطريقة الموصى بها (Render / Railway / Fly.io):**
1. ارفع المشروع إلى مستودع Git ثم اربطه بالمنصة (لا ترفع مجلد المشروع يدويًا).
2. أمر التشغيل: `npm start` ، والمنصة تمرّر `PORT` تلقائيًا وتثبّت الاعتمادات بنفسها.
3. اضبط متغيّرات البيئة: `USER_PASSWORD`, `ADMIN_PASSWORD`, `SESSION_SECRET` (نص عشوائي طويل).
4. **مهم:** اربط قرصًا دائمًا (Persistent Disk/Volume) على المسار `/app/data` حتى لا تُفقد القاعدة عند إعادة التشغيل. المنصة توفّر HTTPS تلقائيًا.

**عبر Docker (أي استضافة حاويات):** المشروع فيه `Dockerfile` جاهز — `docker build -t hashimi-bills . && docker run -p 8765:8765 -v $PWD/data:/app/data --env-file .env hashimi-bills`.

**بخصوص رسالة الرفع** «TypeScript files were found / use wrangler»: مشروعنا **JavaScript خالص بلا أي خطوة بناء**؛ الرسالة سببها أن أداة الرفع تفحص مجلد `node_modules` (الذي يحوي ملفات `.d.ts` من مكتبات مثل Express). الحل: **لا ترفع `node_modules`** — انشر عبر Git (المنصة تثبّت الاعتمادات) أو عبر `Dockerfile` (يستبعده `.dockerignore`). 
> ملاحظة: استضافات «الملفات الثابتة» أو Cloudflare Pages/Workers لا تشغّل خادم Node + SQLite + SSE كما هو؛ استخدم استضافة Node (مثل أعلاه).

## البنية (فصل الطبقات)
```
server/   الخادم: index.js (Express) · db.js (SQLite) · auth.js (مصادقة)
          · routes/ (invoices, activity, admin) · changeTracker.js (الخوارزمية) · sse.js (المزامنة)
public/   الواجهة: index.html · styles.css · js/ (api, auth/main, invoices, bell, admin, realtime, notifications)
data/     قاعدة البيانات + النسخ (لا تُرفع إلى Git)
```

## ملاحظات أمان
- كلمات السر تبقى في الخادم (`.env`/متغيرات البيئة) ولا تظهر في الواجهة إطلاقًا.
- نموذج كلمة السر المشتركة مناسب لفريق صغير موثوق؛ يمكن ترقيته لحسابات فردية لاحقًا.
- للنشر العام استخدم HTTPS (توفّره المنصة) واجعل `SESSION_SECRET` قويًا.
